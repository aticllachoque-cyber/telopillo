# Create Initial Documentation

Help me create the initial documentation structure for a new feature or task.

## Documentation Structure

Create a folder in `docs/` following this naming convention:
- `docs/TICKET-ID-short-description/` (e.g., `docs/MPEE-1234-feature-name/`)

## Required Documents

Generate the following files in order:

### 1. README.md
- Overview of the feature/task
- Document navigation table
- Quick links (Linear ticket, branch name)
- Problem summary (2-3 sentences)
- Solution summary (2-3 sentences)
- Estimated effort
- Key files table

### 2. PRD.md (Product Requirements Document)
- Problem statement with context
- Goals and non-goals
- Requirements (functional and non-functional)
- Success criteria
- Out of scope items
- Dependencies and assumptions

### 3. architecture.md
- System overview with Mermaid diagrams
- Component design
- Data flow
- Integration points
- Error handling strategy

### 4. implementation-plan.md
- Prerequisites checklist
- Step-by-step implementation phases
- Testing strategy
- Rollback plan
- Definition of done

## Information Needed

To create this documentation, I need:

1. **Ticket ID** — The Linear ticket reference (e.g., MPEE-1234)
2. **Problem** — What issue are we solving?
3. **Proposed solution** — High-level approach
4. **Scope** — What's included and excluded?
5. **Key files** — Which files will be affected?
