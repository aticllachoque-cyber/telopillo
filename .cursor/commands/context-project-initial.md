# Initial Context Gathering

Before we begin implementation, please take time to thoroughly understand the codebase and project context.

## Exploration Checklist

1. **Explore the relevant code areas** — Examine existing files, folders, and patterns related to this task
2. **Identify established patterns** — Note architectural decisions, naming conventions, coding styles, and common patterns already in use
3. **Check dependencies and integrations** — Look for related systems, modules, or services that might be affected
4. **Review similar implementations** — Find comparable features or functionality that can serve as reference points
5. **Understand the context** — Consider why this task exists, what problem it solves, and how it fits into the larger system

## Share Your Findings

Once you've explored, please share:

- What patterns and conventions you've identified that we should follow
- Any potential concerns, conflicts, or technical considerations
- Questions or clarifications needed before proceeding
- Your proposed approach aligned with the project's existing architecture
